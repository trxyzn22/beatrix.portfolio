Facebook Group Auto Poster (n8n + Playwright)

A Facebook Group posting automation system built with n8n, Playwright, Google Sheets, Node.js, Express.js, and Cloudflare Tunnel. The project was designed to reduce the time spent manually posting inventory listings to Facebook Groups while keeping human control over inventory updates and caption preparation.

Overview

This automation helps improve business productivity by reducing repetitive manual work involved in posting inventory listings to Facebook Groups. Instead of manually creating each post one by one, inventory data, captions, image links, posting schedules, and post status are managed through Google Sheets.

The workflow uses n8n to check scheduled rows, filter posts that are ready and due, and send the post data to a locally hosted Playwright service through Cloudflare Tunnel. The Playwright service opens the Facebook Group, prepares the post composer, downloads the product image, uploads it, and enters the caption.

A human layer was intentionally kept in the process. Inventory and captions are still manually prepared because the units are fast-moving and are usually one-unit listings. This allows the automation to reduce posting time without sacrificing listing accuracy.

Challenges & Solutions

Reducing Manual Facebook Group Posting

Challenge: Posting inventory manually to Facebook Groups takes time, especially when listings need images, captions, schedules, and status tracking.

Solution: Built an n8n workflow that reads scheduled listings from Google Sheets and sends post data to a Playwright automation service.

Keeping Human Control Over Inventory

Challenge: Inventory moves quickly and many listings are usually one unit only, so fully automated caption and inventory generation could lead to inaccurate posts.

Solution: Kept inventory updates and caption preparation as a human-controlled layer while automating the repetitive posting task.

Working Around Facebook Group API Limitations

Challenge: Direct API-based posting to Facebook Groups is limited and not reliable for this use case.

Solution: Used Playwright browser automation with a persistent browser profile to perform posting actions through a real browser session, helping reduce platform risk compared to unstable or unofficial direct API approaches.

Connecting Cloud n8n to a Local Browser Automation Service

Challenge: n8n was hosted remotely, while the Playwright browser automation needed to run locally using the same trusted device and Facebook session.

Solution: Used Cloudflare Tunnel to securely expose the local Express.js endpoint without opening local network ports.

Architecture & System Design

Google Sheets

Stores post data including schedule, group URL, image URL, caption, ID, row number, and posting status.

n8n Scheduler

Runs on a schedule, checks the spreadsheet, filters due posts, and sends post data through an HTTP Request node.

Cloudflare Tunnel

Bridges the hosted n8n workflow to the local Playwright server running on the user's machine.

Express.js Server

Receives post data from n8n through a POST endpoint and passes it to the Playwright automation script.

Playwright

Opens a persistent Facebook browser session, prepares the post composer, uploads the image, and enters the caption.

Facebook Groups

Receives the prepared inventory post, reducing the need for repetitive manual posting.
